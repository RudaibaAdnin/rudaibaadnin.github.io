Overview

LessonA11y is a web-based concept and prototype designed to support blind teachers of students with visual impairments (TVIs) in planning lessons, making materials accessible, and staying informed about software/tools they rely on. This page summarizes the design goals, ideation journey, and prototype implementation.

Focus: Accessibility in lesson planning, material preparation, and technology updates.

Users: Blind TVIs and their collaborators (e.g., Teaching Assistants).

Approach: Storyboarding, low-fi sketches, and an HTML + Bootstrap prototype.

“If I know I have an IEP coming at a certain time, I start a month in advance… If I don’t do it like that, it just hits me, and then I panic, and I don’t get anything done.”

Insights from prior work: Okungu et al., 2019; Brewer et al., 2023.

Design Goals

From the needs of blind TVIs and prior literature, we articulated four design goals (DGs).

DG1

Planning Lessons

Provide an accessible interface to organize lesson plans and materials efficiently. Recommend accessible tools for students with visual impairments, notify users about tool updates, and support requesting assistance from TAs when needed.

DG2

Making Materials Accessible

Automate accessibility processing for documents and media: add alt text, tag PDFs, and assist with image descriptions via AI to reduce prep time and cognitive load.

DG3

Staying Updated on Tools

Provide access to reviews and experiences from other blind TVIs about software updates and new tools so teachers can anticipate issues and integrate technology confidently.

DG4

Communication with TAs

Enable clear task-sharing and communication with teaching assistants grounded in the concept of interdependence, including delegating accessibility tasks for upcoming lessons.

Ideation Process

We storyboarded a typical day for a blind TVI, explored multiple low-fidelity sketches, and synthesized ideas via affinity diagramming. Voice-only concepts were set aside to prioritize robust screen reader workflows now, while keeping room for optional voice features later.

Prototype Implementation

We prototyped LessonA11y with HTML and Bootstrap. The prototype demonstrates three flows: Lesson Planning, Make Materials Accessible, and Learn about Software. Live site: cs7390.netlify.app · Video demo: Google Drive.

Lesson Planning

Add and organize tasks with titles, descriptions, and associated tools/materials. Check for tool updates and accessibility needs (announced via aria-live) and optionally send plans to a TA.

Making Materials Accessible

Generate accessibility reports, auto-create alt text, tag PDFs, or route items to a TA. The prototype showcases a template flow to illustrate interactions and outcomes.

Staying Informed About Software

Read and write reviews from other blind TVIs about software and updates, helping teachers anticipate changes and adopt tools more confidently.
